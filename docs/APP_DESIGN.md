# 柿川亭×アブララボ 経営ダッシュボード 設計書

対象: masa（柿川亭・アブララボ オーナー）向け 日報／会計集計ダッシュボード
配信: サーバーレス（GitHub Pages）／データ取得: Googleスプレッドシート gviz（`docs/DATA_MODEL.md` 準拠）
デザイン方針: `dataviz` スキル準拠（配色は検証済みパレット、light/dark両対応、フォーム選定の原則に従う）

---

## 1. コンセプト＆アプリ名

**コンセプト**: 「開いた瞬間に、今日の店の調子がわかる」。数字を眺めて自分で計算する時間をゼロにする。経営者が知りたい順（今日→今月→傾向→中身→比較）に情報を並べ、1画面目（ホーム）だけで「儲かっているか／異常はないか」に答えを出す。細かい分析はホームから1タップで深掘りできる構成にする。

### 名前案

**案A：「番頭（ばんとう）」** （推し案）
old-school な「番頭さん＝経営者の右腕として毎日帳簿を見てくれる人」のイメージ。無機質な「ダッシュボード」より人格を感じる名前で、「今日も番頭さんに聞いてみるか」と毎日開く習慣に自然につながる。ロゴは油そばの器を持った番頭風の簡易アイコン程度で十分（ながおか探検録のようなフルキャラは不要、経営ツールなので抑えめに）。
- サブタイトル案：「柿川亭・アブララボ 経営番頭」

**案B：「あぶらミル」**
両店共通の看板食材「油」＋「見る（見える化）」の語呂合わせ。柿川亭（油そば）とアブララボ（油ラボ）のブランド一貫性を出しやすく、カジュアルで覚えやすい。ながおか探検録のノリに近いので、masaのブランド全体を統一したい場合はこちらが合う。

どちらも実装難度は同じ。**経営数字を扱う真面目さ寄りなら「番頭」、ブランド遊び心を優先するなら「あぶらミル」**。以降の設計は名前非依存だが、本書では便宜上「番頭」で表記する。

## 2. 情報設計（画面構成）

ナビは深くしない。**5画面＋ホームが中心**。モバイルは下部タブバー、PCは左サイドバー（同じ5項目）。全画面共通ヘッダーに「期間切替」「店舗切替」「最終同期時刻」を常設し、フィルタは画面ごとに持たせない（dataviz原則：フィルタは1箇所で全体をスコープする）。

| # | 画面 | 目的 | 主要素 |
|---|------|------|--------|
| 1 | **ホーム** | 今日・今月の状態を1画面で把握。異常があれば即気づく | Hero（簡易利益）＋KPIカード4枚＋日次推移＋チャネル構成＋今日の速報 |
| 2 | **推移** | 売上・客数・客単価の時系列トレンドを深掘り | 日次/週次/月次の折れ線・棒、曜日別パターン、前月/前年同月比較 |
| 3 | **メニュー分析**（柿川亭のみ） | 何が売れているか、サイズ/トッピングの傾向 | ベストセラーTOP10、カテゴリ構成、トッピング付帯率、メニュー別推移 |
| 4 | **コスト・利益** | 儲かっているかの内訳 | 仕入経費（費目別/仕入先別）、人件費、簡易利益・利益率の推移、費用構成 |
| 5 | **店舗比較** | 柿川亭とアブララボを並べて見る | 売上/客数/客単価/利益の2店舗並列、将来的にアブララボ立ち上げ後の主戦場 |

補足：
- 「メニュー分析」はアブララボにメニュー列が整うまでは柿川亭専用画面として出し、アブララボ側は「まだデータがありません」の空状態表示にする。
- 「仕入経費」「人件費」の伝票検索・明細一覧はPhase2以降、コスト・利益画面内のドリルダウン（テーブル）として提供（別画面にしない＝深追いしすぎない）。

## 4. 主要KPI・分析（優先度つき）

計算式は `docs/DATA_MODEL.md` の列定義に準拠。「簡易利益＝売上－(仕入経費(税込)＋人件費)」「客単価＝売上÷推定来店客数」が基本式。

### P1：ホーム画面に常設

| 指標 | 説明・計算式 | 最適な表現 |
|---|---|---|
| 当日売上／当月累計売上 | 券売機+anydeli+クーポン+通販の合計 | Stat tile（値＋前日比/前月比delta） |
| 当日／当月 推定来店客数 | 券売機杯数+anydeli換算人数+クーポン換算人数 | Stat tile |
| 客単価 | 売上÷推定来店客数 | Stat tile |
| 簡易利益（当月） | 売上－(仕入経費税込+人件費) | Hero figure（符号でdiverging＝黒字は成功色、赤字はcritical色） |
| 総経費率 | (仕入経費+人件費)÷売上 | Stat tile＋閾値超えでstatusチップ（warning/critical、アイコン+ラベル必須） |
| 今月の日次売上推移 | 日次_店舗別を積み上げ | 折れ線（今月＝実線アクセント色、前月＝点線・淡色で同一スケール重ね） |
| チャネル構成（当月） | 券売機/anydeli/クーポン/通販の内訳 | 横棒の積み上げ（4区分・direct label必須） |

### P2：推移／コスト・利益画面

| 指標 | 説明 | 最適な表現 |
|---|---|---|
| 曜日別平均売上 | 曜日ごとの平均・件数 | 棒グラフ（sequential、平日/休日をemphasisで強調も可） |
| 前月・前年同月比較 | 同スケールで指数化(100=基準月) して比較 | 折れ線2本（同一Y軸、絶対に2軸化しない） |
| 仕入経費 費目別・仕入先別 | 仕入経費タブを仕入先・費目で集計 | 横棒（ランキング、上位7＋その他） |
| 人件費率 | 人件費÷売上 | Stat tile＋トレンド折れ線 |
| 簡易利益率の推移 | 簡易利益÷売上 の月次推移 | 折れ線＋0ラインをbaseline表示 |
| 経費構成（仕入 vs 人件費） | 総経費の内訳 | 積み上げ棒 or 100%積み上げ棒（月次） |

### P3：メニュー分析画面（柿川亭）

| 指標 | 説明 | 最適な表現 |
|---|---|---|
| ベストセラーTOP10 | メニュー別純販売数（発行－取消） | 横棒ランキング（単系列＝sequential blue、値をバー先端に直接ラベル） |
| カテゴリ構成 | 並/大/特盛り、焼豚シリーズ、GF、チャーシュー丼等 | 積み上げ棒（分類は設定タブ準拠、8種超は「その他」に畳む） |
| トッピング付帯率 | トッピング合計数÷杯数 | Meter（同系色トラック）または%のStat tile |
| メニュー別日次推移 | 主要メニューの売れ行き変化 | 折れ線（最大3〜4系列、direct label必須。5系列超は小分割表示） |

### P4：店舗比較画面

| 指標 | 説明 | 最適な表現 |
|---|---|---|
| 売上/客数/客単価/利益 の2店舗並列 | 柿川亭とアブララボ | グループ棒（2系列＝色は店舗固定色、direct label） |
| 店舗別利益の黒字/赤字 | 現状アブララボは赤字運用 | Diverging（青=黒字／赤=赤字、灰色ゼロ基準線） |

### P5：将来（Phase4以降）

- 曜日×週のカレンダーヒートマップ（客単価・売上の変動可視化、sequential blue）
- 仕入先別支出トレンド、発注最適化の示唆
- スタッフ別稼働時間・人件費（タイムカードタブ活用）
- 前年同月比アラート（急落検知）

## 5. ビジュアル方針

`dataviz` スキルの検証済みデフォルトパレットをそのまま採用（自前ブランド色を作らず、確定済みの色を使う＝実装後すぐ検証済み品質が出る）。

**配色（light / dark 両対応）**
- 画面地: `#fcfcfb`（light）／ `#1a1a19`（dark）
- 本文/数字インク: `#0b0b0b`（light）／ `#ffffff`（dark）、補助テキスト `#52514e` / `#c3c2b7`
- 店舗識別色（固定・入れ替えない）: **柿川亭＝青**（`#2a78d6` light / `#3987e5` dark）、**アブララボ＝オレンジ**（`#eb6834` / `#d95926`）
- チャネル4区分（券売機/anydeli/クーポン/通販）: アクア→イエロー→マゼンタ→グリーンの順（4系列時はダイレクトラベル必須というルール通り、積み上げ棒に値を直接表示）
- 損益の良し悪し（黒字/赤字、前月比の良化/悪化）: 成功色 `#006300`(light)/`#0ca30c`(dark) と diverging の赤（`#e34948`系）。**status色（good/warning/serious/critical）は必ずアイコン＋ラベル同時表示**、色だけで判断させない
- メニューランキングなど単系列棒: sequential blue（`#cde2fb`→`#0d366b`）1色の濃淡のみ

**タイポ**: システムsans（`system-ui, -apple-system, "Segoe UI", sans-serif`）で統一。Hero figureとKPIカードの数字は比例数字（proportional）、テーブルの数列だけ`tabular-nums`。

**カード/チャートの見せ方**: 角丸カード、`rgba(ink,10%)`の極細ボーダーのみで枠取り（影を多用しない）。バーは最大24px幅・4px角丸（データ端のみ）、線は2pxで丸角、グリッド線はヘアラインの控えめグレー。ホバー時にツールチップ（クロスヘア＋全系列一括表示、値を強調・系列名は補助）。

**モバイルでの畳み方**: KPIカードは2列×2行に折り返し、チャートは常に全幅1列で縦積み。テーブルは横スクロール＋先頭列（日付）をsticky固定。フィルタ行は横スクロール可能な1行のまま増やさない。

**ダークモード**: OS設定に自動追従＋画面内トグルで手動切替も用意（夜間の店舗PCや外出先スマホ利用を想定）。

## 6. 同期・技術の推奨

### データ取得（gviz）

サーバーレスのまま常時同期する構成：

```
https://docs.google.com/spreadsheets/d/1UXr4m54rClrJknIn_o3DRupMQbYRriSXmykAkJkQx90/gviz/tq?tqx=out:json&sheet=<タブ名>
```

- タブ名は`encodeURIComponent`必須（日本語タブ名対応：例 `日次_柿川亭`）。
- レスポンスは`google.visualization.Query.setResponse(...)`で包まれたJSONP風テキストなので、`response.text()`取得後に最初の`(`〜最後の`)`を抜き出してJSON.parse、またはCSV出力（`tqx=out:csv`）でシンプルにパースする方式でも可。**列はヘッダー名で参照**し、列の並び替えに強くする（インデックス固定参照は避ける）。
- 取得対象タブ：`日次_柿川亭` / `日次_アブララボ` / `メニュー_柿川亭` / `メニュー_アブララボ` / `仕入経費` / `人件費` / `設定`。**`月次集計`タブはPhase1では「検算用の参照」に留め、月次の実表示はアプリ側で日次データから都度集計する**（複数月の履歴表示・任意期間の前月比較に対応するため。月次集計タブは対象月が1つしか持てない設計なので、これに依存すると過去月比較ができない）。

### 更新頻度・体験

- ページ読み込み時に即フェッチ。加えて**5分間隔のポーリング**（タブが非アクティブなときは`Page Visibility API`で一時停止し、Google側への無駄な負荷を避ける）。
- 手動更新ボタン／ヘッダーの同期時刻タップでも即再取得。
- **stale-while-revalidate**: `localStorage`に前回取得データをキャッシュし、起動時はキャッシュを即表示→裏で最新を取得して差し替え。空白画面・スケルトンではなく「前回の数字がすぐ出る」体験を優先（dataviz原則の「再取得中はレイアウトを保つ」にも合致）。

### masaに依頼が必要な作業（要事前確認）

1. スプレッドシートの共有設定を「**リンクを知っている全員が閲覧可**」に変更（ファイル→共有）。編集権限は付与しない。
2. **タブ名を変更しない**（変更するとURLが壊れる）。列の並び替え・列名変更も避けたいが、アプリ側はヘッダー名参照にするので多少の並び替えは吸収可能。列名自体の変更は都度連絡してもらう。
3. データの機密性について：GitHub Pagesは基本公開ホスティングのため、URLを知っていれば誰でも閲覧可能。金額データを扱うため、**リポジトリ名・公開URLを外部にリンクしない運用**を徹底し、必要ならPhase2でアプリ側に簡易PIN画面（クライアントサイドのみ・気休め程度）を足すことも検討。真の非公開が必要ならGitHub Pages以外（要バックエンド）への切替を将来検討。

### 確認しておきたい前提（要masa確認）

- `日次_柿川亭`等のタブは**その月だけのデータか、通年で行が積み上がっていくのか**。後者であれば複数月の推移・前年同月比がそのまま作れるが、前者（毎月シートをリセット/差し替え）だと過去データの保持方法を別途決める必要がある。
- `メニュー_柿川亭`の約40列を「カテゴリ構成」に集計するための**メニュー→分類の対応表**が必要（設定タブの分類リストはあるが、列ごとの紐付けは明示されていない）。Phase3着手前にこの対応表を作成する。

## 7. 段階リリース案

**Phase1（MVP）**
- 同期基盤（gviz取得・キャッシュ・最終更新表示・手動更新）
- ホーム画面：当日/当月の売上・客数・客単価・簡易利益・総経費率のKPIカード、前日/前月比delta、今月の日次売上推移（前月重ね）、チャネル構成
- 期間切替（今日/今週/今月/カスタム）・店舗切替（全店/柿川亭/アブララボ）
- レスポンシブ（PC+スマホ）、light/dark対応
- データソース：`日次_柿川亭` `日次_アブララボ` `設定`（`月次集計`は検算用）

**Phase2**
- コスト・利益画面（仕入経費の費目別/仕入先別ランキング、人件費率、簡易利益率の推移）
- 曜日別パターン分析、前月・前年同月の指数比較

**Phase3**
- メニュー分析画面（メニュー→カテゴリ対応表の整備が前提）：ベストセラー、カテゴリ構成、トッピング付帯率
- 店舗比較画面の本格運用（アブララボのデータが揃い次第）

## 実装Phase1の推奨スコープ（まとめ）

ホーム画面1枚に絞る：**当日/当月の売上・客数・客単価・簡易利益・総経費率**をKPIカードで見せ、**今月の日次推移（前月を薄く重ねる）**と**チャネル構成（券売機/anydeli/クーポン/通販）**で「なぜその数字か」を補足する。データソースは`日次_柿川亭` `日次_アブララボ` `設定`のみとし、`メニュー_柿川亭`・`仕入経費`・`人件費`はPhase2以降に回す。gvizのポーリング同期＋stale-while-revalidateキャッシュ、店舗色（柿川亭=青／アブララボ=橙）の固定、light/dark両対応、スマホ2列カードのレスポンシブを最初から組み込む。これだけで「毎朝開けば今日と今月の調子が一目でわかる」状態が完成する。
